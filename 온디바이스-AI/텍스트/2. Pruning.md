## Slide 12

# 2. Pruning - Granularity

## Pattern-based Pruning - N:M sparsity

- Constraint to have at least 2 zeros in length-4 blocks.
- N:M sparsity means that in each contiguous M elements, N of them is pruned.
- A classic case is 2:4 sparsity (50% sparsity).
- Supported by NVIDIA's Ampere GPU Architecture, which delivers up to 2x speed up.

## Diagram Labels

- Dense Matrix
- Structured-sparse matrix W
- Fine-grained structured-sparse matrix format
- `R x C/2 elements + R x C/2 2bits meta data`
- Structured-sparse and compressed matrix W
  - Non-zero data values
  - 2-bits indices

Source note: Adapted from "Machine Learning Hardware and Systems" Lecture (Fall 2024) by Prof. M. Abdelfattah (Cornell Tech).

## Slide 16

# 2. Pruning - Criterion

## Magnitude-based Pruning

- Magnitude-based pruning considers weights with larger absolute values are more important than other weights.
  - For element-wise pruning, `Importance = |W|`

## Example

Weight:

```text
[ 3  -2 ]
[ 1  -5 ]
```

L1-norm, element-wise importance:

```text
[ |3|  |-2| ]
[ |1|  |-5| ]
```

Importance:

```text
[ 3   2 ]
[ 1   5 ]
```

Pruned weight:

```text
[ 3   0 ]
[ 0  -5 ]
```

Source note: Adapted from "TinyML and Efficient Deep Learning Computing" Lecture (Fall 2024) by Prof. Song Han (MIT).

## Slide 17

# 2. Pruning - Criterion

## Magnitude-based Pruning

- A heuristic pruning criterion
  - Magnitude-based pruning considers weights with larger absolute values are more important than other weights.
  - Magnitude is also known as Lp-norm defined as:

```text
||W^(S)||_p = (sum_{i in S} |w_i|^p)^(1/p)
```

where `W^(S)` is a structural set of parameters.

## Example

Weight:

```text
[ 3  -2 ]
[ 1  -5 ]
```

L2-norm, row-wise importance:

```text
row 1: sqrt(13) = sqrt(|3|^2 + |-2|^2)
row 2: sqrt(26) = sqrt(|1|^2 + |-5|^2)
```

Importance:

```text
[ sqrt(13) ]
[ sqrt(26) ]
```

Pruned weight:

```text
[ 0   0 ]
[ 1  -5 ]
```

Source note: Adapted from "TinyML and Efficient Deep Learning Computing" Lecture (Fall 2024) by Prof. Song Han (MIT).

## Slide 18

# 2. Pruning - Criterion

## Scaling-based Pruning

- Pruning criterion for filter pruning
  - A scaling factor is associated with each filter, i.e. output channel, in convolutional layers.
    - The scaling factor is multiplied to the output of that channel.
    - The scaling factors are trainable parameters.
  - The filters/output channels with small scaling factor magnitude will be pruned.

## Example Diagram

Before pruning:

| Filter | Channel Scaling Factor | Activation |
| --- | ---: | --- |
| Filter 0 | 1.17 | Channel 0 |
| Filter 1 | 0.10 | Channel 1 |
| Filter 2 | 0.29 | Channel 2 |
| Filter 3 | 0.82 | Channel 3 |
| Filter N-1 | 0.56 | Channel N-1 |

After pruning:

| Filter | Channel Scaling Factor | Activation |
| --- | ---: | --- |
| Filter 0 | 1.17 | Channel 0 |
| Filter 3 | 0.82 | Channel 3 |
| Filter N-1 | 0.56 | Channel N-1 |

# 2. Pruning - Criterion

## Activation-based pruning

- Observation: If we use ReLU, many activations will be zero.
- Idea: If some output neurons almost always produce zero, we can safely remove the weights connected to it.
- Neuron pruning is coarse-grained weight pruning.

## Matrix Example

Weight:

```text
[ -13  -11   4 ]
[   7    2 -13 ]
[ -29   10  12 ]
```

Input:

```text
[ X1  X2  X3 ]
[ X4  X5  X6 ]
[ X7  X8  X9 ]
```

Activation:

```text
[ 0  0  0 ]
[ 6  8  0 ]
[ 0  2  9 ]
```

- Hu et al., "Network Trimming: A Data-Driven Neuron Pruning Approach towards Efficient Deep Architectures," arXiv 2016

## Slide 20

# 2. Pruning - Ratio

- We need different pruning ratios for each layer since different layers have different sensitivity.
  - Some layers are more sensitive, e.g. first layer.
  - Some layers are more redundant.
- We can perform sensitivity analysis to determine the per-layer pruning ratio.

## The process of Sensitivity Analysis

VGG-11 on CIFAR-10 dataset:

1. Pick a layer in the model.
2. Prune the layer with pruning ratio in `{0, 0.1, 0.2, ..., 0.9}`.
3. Observe the accuracy degrade for each pruning ratio.
4. Repeat the process for all layers.

## Plot

- X-axis: Pruning Rate (Percentage of Weights Pruned Away)
- Y-axis: Accuracy (%)
- Layers: `L0`, `L1`, `L2`, `L3`, `L4`, `L5`
- A threshold `T` is marked around the mid-70% accuracy range.

Source note: Adapted from "TinyML and Efficient Deep Learning Computing" Lecture (Fall 2024) by Prof. Song Han (MIT).

## Slide 21

# 2. Pruning - Regularization

- When training neural networks or fine-tuning quantized neural networks, regularization is added to the loss term to:
  - Penalize non-zero parameters
  - Encourage smaller parameters
- The most common regularization for improving performance of pruning is L1/L2 regularization.

## L1-Regularization

```text
L' = L(x; W) + lambda |W|
```

```text
||w||_1 = |w_1| + |w_2| + ... + |w_N|
```

1-norm is also known as L1 norm.

## L2-Regularization

```text
L' = L(x; W) + lambda ||W||^2
```

```text
||w||_2 = (|w_1|^2 + |w_2|^2 + ... + |w_N|^2)^(1/2)
```

2-norm is also known as L2 norm or Euclidean norm.

## Examples

- Magnitude-based fine-grained pruning applies L2 regularization on weights.
- Network Slimming applies smooth-L1 regularization on channel scaling factors.
