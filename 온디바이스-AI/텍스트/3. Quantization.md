# 3. Quantization - Introduction

## What is Quantization?

Quantization is the process of constraining an input from a continuous or otherwise large set of values to a discrete set.

## Example

- Continuous signal
- Quantized signal
- Quantization error
- Original image
- 16-color image

The difference between an input value and its quantized value is referred to as quantization error.

Source note: Adapted from "TinyML and Efficient Deep Learning Computing" Lecture (Fall 2024) by Prof. Song Han (MIT).

## Benefits of Quantization

### Memory usage

- 8-bit versus 32-bit weights and activations stored in memory.

### Power consumption

- Significant reduction in energy for both computations and memory access.

| Metric | INT8 | FP32 | Reduction |
| --- | ---: | ---: | --- |
| Add energy (pJ) | 0.03 | 0.9 | 30x energy reduction |
| Mult energy (pJ) | 0.2 | 3.7 | 18.5x energy reduction |

Memory access energy:

| Memory | Energy (pJ) |
| --- | ---: |
| Cache (64-bit), 8KB | 10 |
| Cache (64-bit), 32KB | 20 |
| Cache (64-bit), 1MB | 100 |
| DRAM | 1300-2600 |

- Up to 4x energy reduction.

### Latency

- With less memory access and simpler computations, latency can be reduced.

### Silicon area

| Unit | INT8 | FP32 | Reduction |
| --- | ---: | ---: | --- |
| Add area (um^2) | 36 | 4184 | 116x area reduction |
| Mult area (um^2) | 282 | 7700 | 27x area reduction |

Source: Marios Fournakis, TinyML talks: a practical guide to neural network quantization.

## Slide 7

# 3. Quantization - Number Representation

## Integer

### Unsigned Integer

- n-bit range: `[0, 2^n - 1]`
- Example bit pattern: `00110001`
- Value: `2^5 + 2^4 + 2^0 = 49`

### Signed Integer

- Sign-magnitude representation
- n-bit range: `[-2^(n-1) + 1, 2^(n-1) - 1]`
- Both `000...00` and `100...00` represent 0.

### Two's Complement Representation

- n-bit range: `[-2^(n-1), 2^(n-1) - 1]`
- `000...00` represents 0.
- `100...00` represents `-2^(n-1)`.

## Slide 8

# 3. Quantization - Number Representation

## Fixed/Floating-point number

### Fixed-Point Number

- Integer part
- Fraction part
- Decimal point

Example bit patterns show fixed-point values such as `49` and `3.0625`.

### Floating-Point Number

- Sign
- 8-bit exponent
- 23-bit fraction (significand / mantissa)

Formula:

```text
(-1)^sign x (1 + Fraction) x 2^(Exponent - 127)
```

Example:

```text
0.265625 = 1.0625 x 2^-2
```

The exponent bias is 127.

## Slide 11

# 3. Quantization - Linear/Non-linear

## Linear Quantization

Relationship:

```text
r = (q - Z) x S
```

Where:

- `r`: floating-point value
- `q`: integer value
- `Z`: integer zero point
- `S`: floating-point scale

Quantization parameters:

- `Z` allows the real number `r = 0` to be exactly representable by a quantized integer.
- `S` is the scaling quantization parameter.

Source note: Quantization and Training of Neural Networks for Efficient Integer-Arithmetic-Only Inference [Jacob et al., CVPR 2018].

## Slide 12

# 3. Quantization - Linear/Non-linear

## Scale of Linear Quantization

Equations:

```text
r_max = S(q_max - Z)
r_min = S(q_min - Z)
r_max - r_min = S(q_max - q_min)
S = (r_max - r_min) / (q_max - q_min)
```

Example:

```text
S = (2.12 - (-1.08)) / (1 - (-2))
  = 1.07
```

Labels:

- floating-point range: `r_min` to `r_max`
- integer range: `q_min` to `q_max`
- zero point: `Z`
- floating-point scale: `S`

## Slide 13

# 3. Quantization - Linear/Non-linear

## Zero point of Linear Quantization

Equation:

```text
Z = q_min - r_min / S
```

Example:

```text
Z = round(-2 - (-1.08 / 1.07))
  = -1
```

Binary-to-decimal mapping:

| Binary | Decimal |
| --- | ---: |
| 01 | 1 |
| 00 | 0 |
| 11 | -1 |
| 10 | -2 |

## Slide 21

# 3. Quantization - Granularity

## Per-Tensor: Symmetric Linear Quantization on Weights

- `|r|_max = |W|_max`
- Using single scale for whole weight tensor.
- Common failure results from:
  - Large differences, more than 100x, in ranges of weights for different output channels.
  - Outlier weights.
- Solution: Per-Channel Quantization.

Source notes:
- Data-Free Quantization Through Weight Equalization and Bias Correction [Markus et al., ICCV 2019]
- Quantization and Training of Neural Networks for Efficient Integer-Arithmetic-Only Inference [Jacob et al., CVPR 2018]
- Adapted from "Machine Learning Hardware and Systems" Lecture (Fall 2024) by Prof. M. Abdelfattah (Cornell Tech).

## Per-Channel Weight Quantization

Input weight matrix:

```text

[ -0.91   1.92   0     -1.03 ]
```

## Per-Channel Quantization

Per-channel maximum values and scales:

```text
|r|max = 2.09, S0 = 2.09
|r|max = 2.12, S1 = 2.12
|r|max = 1.92, S2 = 1.92
|r|max = 1.87, S3 = 1.87
```

Quantized matrix:

```text
[ 1  0  1  0 ]
[ 0  0 -1  1 ]
[ 0  1  0 -1 ]
[ 1  0  1  1 ]
```

Reconstructed matrix:

```text
[ 2.09  0     2.09  0    ]
[ 0     0    -2.12  2.12 ]
[ 0     1.92  0    -1.92 ]
[ 1.87  0     1.87  1.87 ]
```

Error:

```text
||W - S ⊙ q_W||_F = 2.08
```

## Per-Tensor Quantization

```text
|r|max = 2.12
S = |r|max / qmax = 2.12 / (2^2 - 1 - 1) = 2.12
```

Quantized matrix:

```text
[ 1  0  1  0 ]
[ 0  0 -1  1 ]
[ 0  1  0  0 ]
[ 1  0  1  1 ]
```

Reconstructed matrix:

```text
[ 2.12  0     2.12  0    ]
[ 0     0    -2.12  2.12 ]
[ 0     2.12  0     0    ]
[ 2.12  0     2.12  2.12 ]
```

Error:

```text
||W - S q_W||_F = 2.28
```

# 3. Quantization - When to Consider Quantization?

## Post-Training Quantization (PTQ)

- So far, we have been talking about post-training quantization (PTQ).
- Weights are frozen and do not change; can precompute scale and zero point.
- Activation scale and zero-points can be calibrated using a minibatch of data.

Process:

```text
Neural Network Training -> Quantization (PTQ)
```

Source note: Adapted from "Machine Learning Hardware and Systems" Lecture (Fall 2024) by Prof. M. Abdelfattah (Cornell Tech).

## Slide 24

# 3. Quantization - When to Consider Quantization?

## Quantization-Aware Training (QAT)

- A full precision copy of the weights `W` is maintained throughout the training.
- The small gradients are accumulated without loss of precision.
- Once the model is trained, only the quantized weights are used for inference.

## Simulated/Fake Quantization

```text
W -> S_W q_W = Q(W)
```

Diagram labels:

- weights
- weight quantization

- inputs

- Layer N-1
- Layer N
- Layer N+1
- Forward

- Conv

- ensure discrete-valued weights and activations in the boundaries
- these operations still run in full precision
