## Slide 3. Processing Closer to Memory

### Digital

- Data access energy and latency dominate.
- Data reuse and data compression이 중요하다.

### Near memory

- Computation is still digital.
- Data transfer cost를 줄인다.
- Memory read energy가 여전히 지배적이다.
- 대역폭은 더 높아진다.

### Deep in-memory

- Memory access and computation을 결합한다.
- Mixed-signal computation을 사용한다.
- Significant energy and latency reduction을 기대할 수 있다.
- 대역폭이 가장 높다.

## Slide 4. Processing Near Memory

### Bring compute closer to the memory

- Benefits
  - Increase memory bandwidth
  - Reduce energy per access
- Challenges
  - Cost

### Embedded DRAM and 3D stacked memory

- Embedded DRAM (eDRAM)
  - SRAM보다 higher density
  - DRAM보다 more energy-efficient
- 3D Stacked Memory (DRAM, SRAM)
  - DDR3보다 higher bandwidth
  - Computation closer to memory

## Slide 5. In-Memory Computing: Analog Computation

### 기본 관계

- Conductance = Weight
- Voltage = Input
- Current = Voltage × Conductance
- Sum currents for addition

### 수식

```text
Output = Σ Weight × Input
```

예시:

```text
Input = V1, V2, ...
Filter Weights = G1, G2, ... (conductance)

I1 = V1 × G1
I2 = V2 × G2
I = I1 + I2
  = V1 × G1 + V2 × G2
```

## Slide 6. Processing in Memory / In-Memory Computing

### Weights are stored as charges

- Activations delivered as analog voltages.
- Word line과 bit line의 precharge 회로를 활성화하면 input activation voltage와 stored weights 사이 곱셈을 수행할 수 있다.
- 변환 오버헤드가 존재한다.
  - Analog to digital
  - Digital back to analog

### 도식 의미

- Input activations가 DAC를 통해 analog voltage로 변환된다.
- A MAC array가 analog computation을 수행한다.
- Analog logic이 multiply/shift, psum, ADC 등을 거쳐 output activations를 만든다.

## Slide 7. Processing in Memory / In-Memory Computing

### Embed processing in peripheral or perform compute in analog domain

Benefits:

- Read multiple weights in parallel: high bandwidth
- Perform multiple computations in parallel: high throughput
- Increase density of compute engines: lower capacitance on input delivery
- Reduce number of conversions: sense amplifier만 final accumulated value에 적용

Memory Technology:

- SRAM
- DRAM
- NVM
  - Flash
  - Memristors

## Slide 9. In-Memory Computing: Challenges

### Non-idealities of analog compute

- Non-linearity
- Sensitivity to process variation
- Sensitivity to temperature variation

### Conversion cost

- Analog-to-digital 변환 비용이 존재한다.
- 변환 횟수는 줄어들 수 있지만 각 변환은 더 비싸질 수 있다.

### Array and device constraints

- Bit cell size가 커질 수 있다.
- Peripheral size가 증가해 storage density가 낮아질 수 있다.
- Limited precision
  - 각 storage element, bit cell, device에 저장 가능한 bit 수가 제한된다.
  - 높은 precision을 지원하려면 serial 처리 또는 여러 element를 써야 한다.
  - Negative value 표현을 위해 array를 두 개 써야 할 수 있다.
- Limited array size / scalability

## Slide 10. In-Memory Computing: Challenges

### DNN과 memory array의 mismatch

- Recent DNNs for classification/object detection
  - Weights: Omega mega to omega mega parameters
  - Inputs: 수천~수만 pixel 규모
- Recent arrays (SRAM/ReRAM)
  - Size: 수십~수백 K cells에서 약 1M cells 수준
- DNN 추론에는 대략 100~100,000 cycles가 필요할 수 있다.
  - Latency
  - Energy consumption

### Target

- 가능한 많은 cell을 활용한다.
- Conventional mapping에서는 일부 cell이 사용되지 않는 문제가 있다.

## Slide 12. Near-Sensor Computing: Logical integration

### 핵심 아이디어

- Image sensors와 DNNs 사이의 bidirectional feedback 기반 협업.
- Image sensor가 raw data를 만들고, low-level filtering을 거쳐 task로 전달한다.
- Task 결과가 sensor 쪽으로 다시 feedback되어 sensor parameter를 조절한다.

### Task-driven feedback

- User-defined task
  - Real-time detection and tracking of objects/activities
  - Specific spatial/temporal attributes 반영
- Deep learning algorithm이 task-driven real-time feedback으로 sensor parameters를 제어한다.

### 조절 가능한 control parameters

- Modality control
- Spatial resolution control
- Temporal resolution control

### 예시

- Average bit depth가 증가할수록 accuracy가 증가하는 경향을 보인다.
- Visual/IR sensor, read-out circuits, control circuits/buffers, machine learning engine이 결합된 구조.

참고:

- B. Mudassar, Jong Hwan Ko et al., "A Camera with Brain-Embedding Machine Learning in 3D Sensors", DATE 2019
- B. Mudassar, Jong Hwan Ko et al., "CAMEL: An Adaptive Camera with Embedded Machine Learning Based Sensor Parameter Control", IEEE JETCAS 2019

## Slide 13. Near-Sensor Computing: Physical integration

### 2-D integration

구조:

```text
CMOS Image Sensor -> A/D Conversion -> Memory -> Neural Network Logic
```

- Multiplexed data movement
- Routing complexities

### 3-D integration

구조:

- Image Sensor
- ADC
- Memory
- Logic

특징:

- 각 signal이 dedicated path를 가진다.
- Modules 사이 bandwidth가 향상된다.
- ADCs multiplexed among pixels이면 여전히 2D readout이 필요하다.
- Smaller ADCs가 필요하다.

### 그래프 의미

- 3D integration은 2D integration 대비 throughput/energy 관점에서 이점을 줄 수 있다.

## Slide 14. Towards in-pixel computing

### Sensor output data removal/reduction

- ADC & readout energy를 줄인다.
  - Image sensor 에너지의 약 50%~75%가 관련될 수 있다.
- DAC & ADC energy를 줄인다.
  - IMC에서 필요
- Sensor에서 host까지 data access를 줄인다.

### Compute energy reduction

- Efficient analog MAC vs. digital MAC.
- Analog MAC이 orders of magnitude more efficient할 수 있다.

### Data path 비교

Conventional:

```text
Sensor/pixel -> Photocurrent/Voltage -> Readout/ADC -> DAC -> IMC -> ADC -> Digital accelerator -> Host
```

In-pixel / in-sensor-in-memory:

```text
Sensor/pixel -> Photocurrent/Voltage -> IMC -> Readout/ADC -> Digital accelerator -> Host
```

핵심은 pixel 또는 sensor 가까이에서 연산해 data movement와 변환을 줄이는 것이다.

## Slide 15. Towards in-pixel computing

### Xu et al. (DAC 2020, TCAS-I 2022)

- 1st layer of BNN
- Direct photocurrent MAC without pixel processing
- 1-bit weights
- Fill factor: 9%
- 1~3 pJ/pixel/frame

### Datta et al. (arXiv 2022, 2023)

- 1st convolution layer for object detection and tracking
- In-situ multi-bit/channel dot product
- Re-purposing digital CDS and SS-ADC
  - +/- weights
  - Batch normalization
  - ReLU functionality
- 24× bandwidth reduction
- 3× latency reduction
- 14% energy reduction

### 도식 의미

- Pixel 내부 또는 주변 회로에서 MAC, comparator, pooling, ADC/digital pooling logic을 결합한다.
- Positive/negative weights를 위한 전압 공급과 multi-channel/overlapping stride 처리가 포함된다.
