## Slide 2. From Manual Design to Automatic Design

### 문제의식

- Manual design은 매우 큰 design space를 다룬다.
- 사람이 직접 선택해야 하는 요소:
  - Number of layers
  - Operation at each layer
  - Hyper-parameters of each operation
  - Topology and connectivity

### 핵심

- NAS는 사람이 설계하던 신경망 구조 선택을 자동화한다.
- EfficientNet, MnasNet 등은 accuracy와 MACs/latency trade-off를 고려한 자동 설계 사례로 제시된다.

## Slide 4. Framework

NAS의 목표:

- Search space 안에서 가장 좋은 neural network architecture를 찾는다.
- 관심 objective를 최대화한다.
  - Accuracy
  - Efficiency
  - Latency
  - Model size 등

일반 프레임워크:

```text
Search Space A -> Search Strategy -> Architecture A in A
                                   -> Performance Estimation Strategy
```

## Slide 5. Search Space

- Search space는 candidate neural network architectures의 집합이다.

종류:

- Cell-level
- Network-level

## Slide 6. Cell-level Search Space: Example

- Normal Cell과 Reduction Cell을 조합해 전체 네트워크를 구성한다.
- 같은 cell 구조를 반복해 쌓고, reduction cell은 feature map 해상도를 줄이는 역할을 한다.
- Cell 내부에서는 candidate operation과 connection을 선택한다.

## Slide 7. Cell-level Search Space

RNN controller가 candidate cell을 생성하는 방식:

- 두 input을 찾는다.
- Input transformation operation을 선택한다.
  - convolution
  - pooling
  - identity 등
- 결과를 결합하는 method를 선택한다.

## Slide 8. Cell-level Search Space: Question

가정:

- 두 candidate input이 있다.
- Candidate operation이 3개 있다.
- Operation 하나가 두 hidden state를 결합한다.

질문:

- 새 cell 하나에 가능한 search space 크기는 얼마인가?
- hidden state가 N개, layer가 B개일 때 가능한 NASNet search space는 얼마인가?

제시 답:

```text
(2 x 2 x M x M x H)^B = 4M^2H^B
```

예시:

```text
M = 5, H = 2, B = 5
=> 약 3 x 10^9 candidates in the design space
```

## Slide 9. Network-level Search Space: Depth Dimension

- Network depth, 즉 layer/block 반복 횟수를 search dimension으로 둔다.
- 예시 depth 후보:
  - `[2, 3]`
  - `[2, 3, 4]`
  - `[2, 3, 4, 5]`
  - `[1, 2, 3]`

## Slide 10. Network-level Search Space: Resolution Dimension

- Input resolution 또는 feature map resolution을 search dimension으로 둔다.
- 예시 후보:
  - `[128, 128]`
  - `[160, 160]`
  - `[192, 192]`
  - `[224, 224]`
  - `[256, 256]`
- Resolution이 커질수록 입력 정보는 많아지지만 연산량과 latency가 증가한다.

## Slide 11. Network-level Search Space: Width Dimension

- Channel width, 즉 각 stage의 channel 수를 search dimension으로 둔다.
- 예시 후보:
  - `[48, 64, 96]`
  - `[192, 256, 384]`
  - `[384, 512, 768]`
  - `[640, 1024, 1600]`
  - `[1280, 2048, 3200]`
- Width가 커지면 표현력은 증가하지만 parameter, FLOPs, memory가 증가한다.

## Slide 12. Network-level Search Space: Kernel Size Dimension

- Kernel size도 search dimension이 될 수 있다.
- Depthwise convolution을 사용하는 모델에서는 각 depthwise convolution의 kernel size를 선택할 수 있다.
- 예시:
  - 3x3
  - 5x5
  - 7x7

## Slide 13. Network-level Search Space: Topology Connection

- 각 path는 search space 안의 하나의 network architecture에 대응한다.
- 이 formulation으로 대표적인 manual design도 표현할 수 있다.
  - DeepLabV3
  - UNet
  - Stacked Hourglass
- Connection topology 자체도 탐색 대상이 된다.

## Slide 14. Search Strategy

Search strategy는 search space를 어떻게 탐색할지 정의한다.

탐색 전략:

- Grid search
- Random search
- Reinforcement learning
- Gradient descent
- Evolutionary search

## Slide 15. Grid Search

- Grid search는 hyper-parameter optimization의 전통적인 방법이다.
- 전체 design space를 각 dimension design space의 Cartesian product로 표현한다.
  - 예: resolution in `[1.0x, 1.1x, 1.2x]`
  - width in `[1.0x, 1.1x, 1.2x]`
- 각 candidate network의 accuracy를 얻기 위해 from scratch로 학습한다.

### 예시 표

| Width / Resolution | 1.0x | 1.1x | 1.2x |
|---|---:|---:|---:|
| 1.0x | 50.0% | 53.0% | 54.9% |
| 1.1x | 51.0% | 53.5% | 55.4% |
| 1.2x | 52.0% | 54.1% | 56.2% |

- 표의 숫자는 candidate network의 accuracy이다.
- 하늘색: latency constraint를 만족.
- 분홍색: latency constraint를 위반.

## Slide 16. Grid Search: Compound Scaling

- EfficientNet은 starting network에 depth, width, resolution의 compound scaling을 적용한다.
- Grid search로 alpha, beta, gamma 값을 탐색한다.
- 새 모델의 total FLOPs가 원본 네트워크의 약 2x가 되도록 한다.

관계:

```text
depth: d = alpha^phi
width: w = beta^phi
resolution: r = gamma^phi
s.t. alpha * beta^2 * gamma^2 ≈ 2
alpha >= 1, beta >= 1, gamma >= 1
```

## Slide 17. Random Search

- Grid search는 정해진 격자 전체를 훑는다.
- Random search는 search space에서 candidate를 무작위로 샘플링한다.
- Dimension별 중요도가 다를 때 random search가 grid search보다 효율적일 수 있다.

## Slide 18. Reinforcement Search

- Neural architecture design을 sequential decision-making problem으로 모델링한다.
- Controller를 RNN으로 구현하고, reinforcement learning으로 학습한다.

### 흐름

```text
Controller (RNN)
-> sample architecture A with probability p
-> train child network with architecture A
-> get accuracy R
-> compute gradient of p and scale it by R
-> update controller
```

### Policy gradient algorithm

- Function parameterized with parameters theta: Deep Neural Network (RNN)
- Action: DNN architecture parameters
- State space: all possible DNNs in search space
- Reward: accuracy, used to update theta

## Slide 19. Gradient-based Search: SuperNet

- SuperNet은 가능한 candidate operation을 모두 포함한 큰 network로 표현된다.
- 각 edge 또는 node에 여러 operation 후보가 연결되어 있다.
- 탐색 과정에서 어떤 operation을 남길지 학습한다.

## Slide 20. Gradient-based Search: SuperNet

### Mixed Operation

후보 operation:

- `conv3x3`
- `conv7x7`
- `avgpool`

각 후보 operation에는 architecture parameter가 붙는다.

- alpha1
- alpha2
- alpha3

Mixed operation은 softmax weighting으로 여러 후보 operation을 섞는다.

```text
o_bar^(i,j)(x) =
Σ_{o in O} exp(alpha_o^(i,j)) / Σ_{o' in O} exp(alpha_o'^(i,j)) * o(x)
```

## Slide 21. Gradient-based Search

- 학습 중 architecture parameter alpha를 최적화한다.
- 학습 후에는 각 edge에서 가장 큰 alpha를 가진 operation만 남긴다.
- 즉, SuperNet에서 discrete architecture를 추출한다.

## Slide 22. Evolutionary Search

- Biological evolution process를 모방한다.

구성 요소:

- Evolution pool
- Fitness evaluation
- Population initialization
- Mutation
- Crossover
- Survivor selection
- Termination and return best

절차 예:

- 가장 오래된 개체를 제거한다.
- 좋은 genotype을 변형해 새 genotype을 만든다.
- 새 genotype을 evolution pool에 삽입한다.

## Slide 23. Evolutionary Search: Fitness Function

- Fitness function은 accuracy와 efficiency를 함께 고려한다.

예시 흐름:

```text
Original Network -> Sample -> Sub Network -> Latency/Accuracy Feedback
```

- Latency constraint를 만족하면 keep architecture.
- Constraint를 위반하면 remove architecture.

## Slide 24. Evolutionary Search: Mutation

### Mutation on depth

- Stage 1 depth: 3 -> Stage 1 depth: 2
- Stage 2 depth: 3 -> Stage 2 depth: 4

즉, stage별 mini block 개수를 바꾸는 mutation이다.

## Slide 25. Evolutionary Search: Mutation

### Mutation on operator

- Depth는 유지한 채 특정 mini block의 operator를 바꾼다.
- 예시에서는 노란색 block 일부가 다른 operator 후보로 교체된다.

## Slide 26. Evolutionary Search: Crossover

- Parent 1과 Parent 2의 block/operator를 조합해 child architecture를 만든다.
- Idea:
  - 각 stage의 operator를 parent들 중 하나에서 randomly choose한다.
- 결과 architecture는 일부 stage/block을 parent 1에서, 일부를 parent 2에서 가져온다.

## Slide 27. Performance Comparison

비교 대상:

- Evolution
- RL
- RS (Random Search)

그래프:

- X축: Experiment Time (hours)
- Y축: Top Testing Accuracy
- Evolution이 빠르게 높은 accuracy에 도달한다.
- RL은 그 다음 수준으로 개선된다.
- Random Search는 상대적으로 낮은 accuracy에 머무른다.
