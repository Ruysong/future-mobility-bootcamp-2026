## 2. 임베디드 시스템과 ECU

임베디드 시스템은 특정 목적을 수행하는 하드웨어와 소프트웨어의 결합이다. 차량에서는 램프, 모터, ADAS, 인포테인먼트처럼 기능별 시스템이 ECU를 통해 제어된다.

ECU(Electronic Control Unit)는 차량의 엔진, 변속기, 안전 시스템, 편의 기능 등을 제어하는 장치이다. 일반적으로 MCU, 메모리, 입출력 장치, 전원 공급 장치로 구성된다.

### Multi-Core 필요성

ADAS와 인포테인먼트처럼 복합 Application이 도입되면서 Data Traffic과 Bandwidth 요구가 커진다. 단순히 Clock Frequency를 올리면 발열과 전력 소모가 커지므로 Multi-Core MCU와 고속 네트워크가 중요해진다.

## 4. IVN과 기존 차량 네트워크

IVN(In-Vehicle Network)은 차량에 설치된 ECU 간 통신을 위해 구성된 차량 내부 네트워크이다. 주요 통신 프로토콜은 CAN(FD), FlexRay, Ethernet, LIN, MOST가 있다.

### CAN의 특징과 한계

CAN은 다음 특징을 가진다.

- Message-Oriented 통신
- Non-destructive bit-wise arbitration
- Multi-Master 구조

CAN limitation:

- 낮은 최대 전송 속도
- Payload 크기 제한
- 대규모 네트워크 확장성 제한
- 보안 기능 미흡

영역 기반 아키텍처로 성장하면서 데이터량이 크거나 보안, 대규모 확장성이 중요한 시스템에는 CAN만으로 한계가 있다.

### MOST의 특징과 한계

MOST(Media Oriented Systems Transport)는 자동차 인포테인먼트와 실시간 멀티미디어 전송에 사용된다.

- POF 또는 전기식 UTP
- 동기식 직렬 전송
- 25/50/150 Mbps 속도
- Ring topology
- Synchronous channel: CD 품질 오디오/영상
- Asynchronous channel: 파일, 패킷, Ethernet(MOST150)

한계:

- 대역폭 확장 한계
- 토폴로지 제약
- IP/Ethernet native 미지원

## 5. Automotive Ethernet

Automotive Ethernet은 차량 내부 고대역폭 통신과 backbone 구조를 위해 발전했다.

주요 흐름:

- 1990년대: LAN Ethernet IEEE 802.3
- 2004년: AUTOSAR Ethernet Standard Protocol 개발
- 2008년: BMW가 industrial Ethernet 사용
- 2011년 이후: BroadR-Reach, OPEN Alliance 중심으로 automotive Ethernet 표준화

주요 규격:

| 규격 | 속도 | 케이블/토폴로지 | 용도 |
| --- | --- | --- | --- |
| 10BASE-T1S | 10 Mbps | 단일 UTP, Multidrop Bus | CAN 대체 후보 |
| 100BASE-T1 | 100 Mbps | 단일 UTP, P2P | 카메라, 파킹 ECU, Gateway |
| 1000BASE-T1 | 1 Gbps | 단일 UTP, P2P | Radar, Lidar, ECU Backbone |
| 2.5/5/10G-T1 | 2.5-10 Gbps | 단일 UTP, P2P | Domain Controller, SDV |

차량에서는 RJ-45 기반 4-pair Ethernet보다 Single-Pair Ethernet이 무게, 부피, 커넥터 크기, EMI/EMC, 비용 측면에서 유리하다.

## 6. OSI 7계층과 Ethernet 위치

OSI 7계층은 통신 과정을 7개의 논리 단계로 나눈 모델이다.

| 계층 | 예시 |
| --- | --- |
| Application | HTTP, FTP, DNS |
| Presentation | SSL, IMAP, MPEG, JPEG |
| Session | Socket, API |
| Transport | TCP, UDP |
| Network | IPv4, IPv6 |
| Data-Link | Ethernet MAC |
| Physical | Ethernet PHY Chip |

CAN은 Data-Link Layer의 CAN Protocol과 Physical Layer의 CAN Physical Layer를 중심으로 설명된다. Ethernet은 Ethernet MAC이 Data-Link, Ethernet PHY가 Physical 계층을 담당한다.

## 7. Ethernet MAC/PHY 구조

Ethernet MAC Module은 MCU 내부에 위치하고 Ethernet frame을 생성한다. PHY Chip은 binary/symbolic data를 electrical signal로 변환해 physical link로 보낸다.

주요 인터페이스:

- `MII`: Medium Independent Interface, 100 Mbps, 25 MHz, 4-bit
- `RMII`: Reduced MII, 100 Mbps, 50 MHz, 2-bit
- `MDIO`: Management Data Input/Output, PHY chip management
- `MDI`: Medium Dependent Interface, RJ45/physical link 쪽 연결

PHY 변환 과정:

- 4B3B/3B2T Encoding
- PAM3 / Echo Cancellation
- Bit stream timing
- Preamble & SFD(Start of Frame Delimiter)

## 8. Ethernet Frame 구조

Ethernet frame은 다음 필드로 구성된다.

| 필드 | 크기 | 의미 |
| --- | --- | --- |
| Preamble | 7 bytes | 수신 동기화 |
| SFD | 1 byte | Start of Frame Delimiter |
| Destination MAC | 6 bytes | 목적지 MAC 주소 |
| Source MAC | 6 bytes | 출발지 MAC 주소 |
| Type/Length | 2 bytes | EtherType 또는 Length |
| Payload | 46-1500 bytes | 상위 계층 데이터 |
| FCS | 4 bytes | 오류 검출 |
| Interframe Gap | 12 bytes | 프레임 간 간격 |

### MAC Address

MAC 주소는 6 byte이며, 앞 3 byte는 OUI(Organization Unique Identifier), 뒤 3 byte는 개인 MAC 번호이다.

- 첫 octet의 최하위 bit가 `0`: Unicast
- 첫 octet의 최하위 bit가 `1`: Multicast
- Broadcast: `FF:FF:FF:FF:FF:FF`

### EtherType

Type/Length 필드가 1500보다 작으면 IEEE 802.3 Length로 해석하고, 1536보다 크면 Ethernet II EtherType으로 해석한다.

### Payload와 Padding

Ethernet frame은 최소 전송 크기 64 bytes 조건을 만족해야 한다. Header 크기는 DA 6B + SA 6B + EtherType 2B + FCS 4B = 18B이다. VLAN tag가 없으면 최소 payload는 46 bytes, VLAN tag가 있으면 42 bytes이다. 실제 payload가 더 작으면 padding octet이 추가된다.

## 9. Hardware Configuration

RA6M3 Ethernet 구조는 다음 모듈을 포함한다.

- ETHERC Module: Ethernet MAC Controller
- EDMAC Module: Ethernet DMA Controller
- PTPEDMAC: Precision Time Protocol 관련 DMA
- Ethernet PHY Chip
- Descriptor & Buffer

### Ethernet PHY Chip Management

PHY-LSI의 MII/RMII register 접근에는 PIR register를 사용한다. MII/RMII management frame 형식의 serial data가 `ET0_MDC`, `ET0_MDIO` pin을 통해 송수신된다.

### EDMAC Descriptor/Buffer

Ethernet DMA Controller는 Ethernet transmit/receive descriptor와 transmit/receive buffer를 사용한다.

Descriptor에는 frame buffer 길이와 frame buffer base address(pointer)가 담긴다. Ethernet frame buffer는 16-byte boundary를 기준으로 배치된다.

## 13. Background B - Unicast, Multicast, Broadcast

| 방식 | 의미 |
| --- | --- |
| Unicast | 네트워크의 한 지점에서 다른 한 지점으로 1:1 전송 |
| Multicast | 특정 그룹에 대한 one-to-many 또는 many-to-many 전송 |
| Broadcast | 모든 수신자에게 동시에 전송 |

Data-link layer에서는 multicast가 broadcast와 유사하게 모든 관련 노드로 확산될 수 있다.

### Multicast 주소 판별

MAC 주소 첫 octet의 최하위 bit가 `1`이면 multicast이다.

예제에서 multicast가 아닌 주소는 다음 2개이다.

- `64:13:C5:62:33:F1`
- `2C:FF:15:32:11:FF`

## 14. VLAN Extension Mode

VLAN(Virtual LAN)은 data-link layer에서 network를 논리적으로 분리하는 partial broadcast domain이다. 같은 물리 switch에 직접 연결되어 있지 않아도 host를 논리 그룹으로 묶을 수 있다.

### IEEE 802.1Q 필드

| 필드 | 설명 |
| --- | --- |
| TPID | Tag Protocol Identifier, VLAN frame 식별 값 `0x8100` |
| PCP | Priority Code Point, 3-bit priority |
| VID | VLAN Identifier, 12-bit VLAN ID |

VID 범위는 `0x000-0xFFF`이며, `0x000`은 VLAN ID 없이 priority만 지정하는 reserved value로 사용된다. `0x001`은 network management VLAN으로 reserved되는 경우가 많다.

### VLAN을 쓰는 이유

- 네트워크 분리로 보안 강화
- Broadcast domain 축소
- 불필요한 traffic 격리로 대역폭/성능 향상
- PCP 우선순위 설정으로 실시간/TSN traffic 보호
- 노드 이동, 추가, 변경 시 물리 재배선 없이 VLAN tag만 변경
- 장비 증설 없이 가상 네트워크 구축

## 16. Non-VLAN Mode

### Unicast

Non-VLAN에서 unicast로 전송하면 switch의 MAC address table에 따라 Destination MAC과 일치하는 한 port로만 message가 전송된다.

실습 예시:

- Source: ECU1
- Destination: ECU4
- EtherType: `0x0A03`
- 결과: ECU4의 LED state만 변화

### Multicast

Non-VLAN에서 multicast로 전송하면 switch MAC address table과 관계없이 모든 port에 동일 message가 전송된다.

실습 예시:

- Source: ECU5
- Destination: ECU2 형태로 multicast bit 설정
- 결과: 모든 ECU의 LED state 변화

## 17. VLAN Extension Mode

### Input 조건

Port가 switch로 frame을 들여보낼 때, 해당 port가 허용하는 VLAN ID에 해당하는 message만 switch로 전송할 수 있다.

예시:

- Port 4 ECU는 VLAN ID 2, 6에 해당하는 message만 전송 가능
- Port 6 ECU는 VLAN ID 2, 8, 9에 해당하는 message만 전송 가능

### Output 조건

Switch가 port로 frame을 내보낼 때는 VLAN ID별 output port membership을 따른다.

| VLAN ID | 이름 | Port |
| --- | --- | --- |
| 2 | Diagnostic | 1, 2, 3, 4, 5, 6, 7, 8 |
| 3 | Video | 1, 2, 3, 8 |
| 4 | Laser | 1, 2, 8 |
| 5 | Audio | 1, 3, 8 |
| 6 | Engine | 1, 4, 5, 8 |
| 7 | Airbag | 1, 5, 8 |
| 8 | Trunk | 1, 6, 8 |
| 9 | Window | 1, 6, 8 |

Unicast는 VLAN membership이 맞는 port 중 Destination MAC이 일치하는 장치로만 전송된다. Multicast는 해당 VLAN에 포함된 port 전체로 전송된다.

### VLAN/Unicast 실습

예시:

- Source: ECU5
- Destination: ECU4
- VLAN ID: 6
- Payload: `sDxl150`
- 결과: ECU4 Servo Motor만 150도로 회전

해석:

- Input: ECU5는 VLAN ID 2, 6, 7 전송 가능
- Output: VLAN ID 6은 port 1, 4, 5, 8로만 전송
- Unicast: 가능한 port 중 Destination MAC이 일치하는 ECU4에만 전송

### VLAN/Multicast 실습

예시:

- Source: ECU2
- Destination: ECU3에 multicast bit 설정
- VLAN ID: 3
- Payload: `sDxl030`
- 결과: ECU1, ECU3 Servo Motor만 30도로 회전

해석:

- Input: ECU2는 VLAN ID 2, 3, 4 전송 가능
- Output: VLAN ID 3은 port 1, 2, 3, 8로 전송
- Multicast: 해당 VLAN에 포함된 전체 port로 동일 message 전송

## 18. VLAN 예제 문제

### 문제 1

ECU2가 VID4, multicast로 message를 보내면 ECU5는 수신할 수 있는가?

정답: 수신할 수 없다. VID4는 ECU1, ECU2, ECU8만 망이 구성되어 있으므로 ECU5는 포함되지 않는다.

### 문제 2

ECU3이 VID6, multicast로 message를 보내면 ECU5는 수신할 수 있는가?

정답: 수신할 수 없다. VID6에는 ECU1, ECU4, ECU5, ECU8만 포함되어 있으나, ECU3은 VID6으로 message를 switch에 입력할 수 없다.

### Backbone 문제

SW1의 port 2에 연결된 ECU2가 VID4, multicast로 message를 보내면 SW2 port 1 ECU1과 SW3 port 1 ECU1이 수신할 수 있는가?

정답:

- SW2 ECU1: 수신 가능
- SW3 ECU1: 수신 불가

VID4는 backbone switch 기준 port 1, 2, 8에 해당하므로 SW1과 SW2 쪽에는 전달되지만 SW3 쪽에는 전달되지 않는다.

## 19. 핵심 요약

- 차량용 Ethernet은 CAN/MOST의 대역폭과 확장성 한계를 보완하기 위해 등장했다.
- Ethernet frame은 MAC 주소, EtherType, Payload, FCS 등 필드 단위로 분석할 수 있다.
- RA6M3의 Ethernet 통신은 MAC Controller, DMA Controller, PHY Chip, descriptor/buffer가 함께 동작한다.
- Raw Ethernet 실습에서는 EtherType `0x0A01`로 PC와 RA6M3 간 frame 송수신을 확인했다.
- Gateway 실습에서는 EtherType `0x0A02`로 Ethernet payload를 CAN frame으로 분할 전송했다.
- VLAN 실습에서는 EtherType `0x0A03`과 802.1Q VLAN ID를 이용해 switch port forwarding 범위를 제어했다.
- Unicast는 Destination MAC이 일치하는 단일 port로, multicast는 VLAN membership에 포함된 여러 port로 전달된다.

## 20. 면접에서 말하기 좋은 포인트

- Automotive Ethernet을 단순히 "빠른 통신"이 아니라 backbone, TSN, 보안, 고대역폭 센서 데이터 처리를 위한 차량 내부 네트워크 전환으로 설명할 수 있다.
- Ethernet MAC/PHY, MII/RMII, MDIO/MDI 관계를 MCU와 PHY chip 연결 구조로 설명할 수 있다.
- Wireshark에서 Ethernet frame의 DA, SA, EtherType, Payload를 직접 확인했고 RA6M3 TFT-LCD 출력과 대조했다.
- Ethernet-CAN gateway에서 payload 크기 차이 때문에 Ethernet payload를 CAN 8-byte data field로 분할해야 함을 확인했다.
- VLAN에서는 input membership, output membership, unicast/multicast 조건이 모두 맞아야 원하는 ECU에만 제어 message가 전달된다는 점을 실습으로 확인했다.
