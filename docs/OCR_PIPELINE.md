# OCR Pipeline

이 저장소의 교안 텍스트는 단순 PDF/PPTX 텍스트 추출이 아니라, 페이지나 슬라이드를 이미지로 렌더링한 뒤 OCR로 다시 읽어 생성한다.

## 목적

- 그림, 표, 도표, 캡처 이미지 안에 들어간 글자까지 텍스트화한다.
- GitHub에서 바로 검색하고 읽을 수 있도록 과목별 `텍스트/` 폴더의 `.md` 파일을 갱신한다.
- 원본 교안은 `교안/`에 그대로 보존하고, 별도 이미지 정리본은 만들지 않는다.

## 처리 방식

1. PDF 교안
   - `pypdfium2`로 각 페이지를 PNG 이미지로 렌더링한다.
   - Windows OCR API를 한국어 인식 엔진(`ko`)으로 실행한다.
   - OCR 결과를 페이지 단위 Markdown으로 저장한다.

2. PPTX 교안
   - PowerPoint COM 자동화로 각 슬라이드를 PNG 이미지로 내보낸다.
   - Windows OCR API를 한국어 인식 엔진(`ko`)으로 실행한다.
   - OCR 결과를 슬라이드 단위 Markdown으로 저장한다.

3. Markdown 출력
   - 출력 위치는 각 과목의 `텍스트/` 폴더다.
   - 파일명은 원본 교안 파일명과 맞춘다.
   - 이미지는 저장하지 않고 OCR로 읽은 순수 텍스트만 저장한다.

## 실행

저장소 루트에서 PowerShell로 실행한다.

```powershell
.\tools\renew-ocr-texts.ps1
```

PowerPoint 자동화가 필요한 PPTX 처리에서는 일반 sandbox 권한보다 실제 데스크톱 세션 권한이 필요할 수 있다.

## 한계

- OCR 특성상 수식, 첨자, 표 구조, 화살표 흐름은 완벽하게 복원되지 않을 수 있다.
- 영어 전용 교안도 현재 Windows에 설치된 한국어 OCR 엔진으로 처리한다. 한국어 엔진은 영문도 인식하지만, 영어 OCR 언어팩이 있으면 품질이 더 좋아질 수 있다.
- 도표의 의미 설명은 생성하지 않는다. 이 파이프라인은 원문 이미지의 글자를 텍스트로 읽는 데 집중한다.
